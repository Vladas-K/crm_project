# Event Agency CRM

CRM-система для event-агентства, созданная на Django, Bootstrap и серверных шаблонах.

## Обзор проекта

Это кастомная CRM для event-агентства. Проект покрывает весь рабочий цикл: от входящих лидов и воронки продаж до ведения мероприятий, подрядчиков, финансов и аналитики.

Интерфейс построен на Django Templates и Bootstrap. Текущая версия уже включает демо-данные и CRUD-экраны для ключевых сущностей без необходимости работать через Django admin.

## Возможности

- управление лидами
- воронка продаж
- работа с клиентами
- мероприятия как центральная сущность системы
- форматы мероприятий с переиспользуемыми шаблонами
- подрядчики
- пакеты услуг
- календарь
- дашборд аналитики
- роли команды и флаги доступа
- загрузка демо-данных
- CRUD-формы без admin для ключевых сущностей

## Модули CRM

- `Leads`
  Входящие заявки с источником, контактными данными, менеджером, этапом воронки, вероятностью сделки и причиной отказа.
- `Pipeline`
  Отдельные этапы воронки: Новый лид, Связались, Бриф, КП отправлено, Переговоры, Договор, Оплата, Потерян.
- `Clients`
  Квалифицированные лиды, переведённые в B2B/B2C клиентов с сегментом, LTV и предпочтениями.
- `Events`
  Центральная сущность, содержащая данные о мероприятии, бюджете, задачах, коммуникациях, документах, рисках и итогах.
- `Event Formats`
  Динамические шаблоны, автоматически создающие задачи, тайминг, бюджетные статьи и структуру подрядчиков.
- `Vendors`
  Внешние подрядчики с ролями, ценами, надёжностью и чёрным списком.
- `Service Packages`
  Готовые пакеты услуг для быстрого формирования коммерческих предложений.
- `Analytics`
  Конверсия, средний чек, прибыль, разбивка по источникам и загрузка команды.

## Встроенная логика

- автоматическое назначение `Sales Manager` для новых лидов
- контроль лидов без ответа более 24 часов
- автоматическое создание задач, тайминга, бюджета и подрядчиков из шаблона формата мероприятия
- предзаполненные стандартные этапы воронки

## Установка

```bash
git clone https://github.com/Vladas-K/crm_project.git
cd crm_project
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py load_demo_data
python manage.py runserver
```

## Запуск проекта

После запуска development-сервера откройте:

```text
http://127.0.0.1:8000/
```

## Демо-данные

В проекте есть management command для загрузки демо-пользователей, лидов, клиентов, мероприятий, подрядчиков, форматов и пакетов:

```bash
python manage.py load_demo_data
```

Сейчас демо-данные включают:

- sales managers
- project manager
- finance user
- admin user
- лиды
- клиентов
- форматы мероприятий
- подрядчиков
- пакеты услуг
- мероприятия со связанными бизнес-данными

## Основные команды

```bash
python manage.py runserver
python manage.py migrate
python manage.py load_demo_data
python manage.py createsuperuser
```
